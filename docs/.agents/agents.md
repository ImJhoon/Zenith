# Project Agent Guidelines

이 문서는 프로젝트에 참여하는 모든 AI 에이전트(개발, 디자인 스킬 포함)가 준수해야 할 전역 규칙과 디자인 시스템 가이드라인을 정의합니다.

## 1. Core Tech Stack
- **Styling**: Tailwind CSS (독점 사용, Bootstrap 등 타 프레임워크 혼용 금지)
- **UI/UX Design**: `frontend-design` 스킬 가이드라인 준수

## 2. Design Philosophy (디자인 철학)
일반적이고 흔한 "AI 생성 스타일(AI Slop)"을 지양하고, 독창적이며 완성도 높은 프로덕션 수준의 디자인을 추구합니다.
- **Bold Aesthetics**: 뚜렷한 컨셉(미니멀리즘, 맥시멀리즘, 글래스모피즘 등)을 정하고 과감하게 표현합니다.
- **Micro-interactions**: 호버, 클릭, 스크롤 시 부드럽고 흥미로운 애니메이션(Tailwind의 `transition`, `animate` 활용)을 적극적으로 추가합니다.
- **Typography**: 기본 시스템 폰트(Arial 등) 사용을 피하고, 프로젝트 성격에 맞는 매력적인 웹 폰트(Google Fonts 등)를 활용합니다.
- **Space & Composition**: 여백을 충분히 활용하여 시각적 답답함을 없애고, 필요에 따라 비대칭 레이아웃 등을 활용하여 단조로움을 탈피합니다.

## 3. Tailwind CSS Usage Rules
- **일관성**: 인라인 스타일(style 속성) 사용을 엄격히 금지하며, 모든 스타일링은 Tailwind 유틸리티 클래스로 처리합니다.
- **테마 확장**: 반복적으로 사용되는 고유한 색상이나 간격은 임의의 값(`w-[13px]` 등)을 남발하기보다 `tailwind.config.js`에 테마로 등록하여 사용합니다.
- **반응형 디자인**: 모바일 우선(Mobile-first) 접근 방식을 사용하여 `sm:`, `md:`, `lg:` 등의 브레이크포인트를 필수로 적용합니다.

## 4. Development Workflow
- **Semantic HTML**: 웹 접근성(a11y)과 SEO를 고려하여 시맨틱 태그(`<header>`, `<main>`, `<article>`, `<section>` 등)를 사용합니다.
- 컴포넌트나 페이지를 구현할 때는 항상 사용자 경험(UX) 관점에서 가장 매력적으로 보일 수 있는 상태를 고민하고 제안해야 합니다.
